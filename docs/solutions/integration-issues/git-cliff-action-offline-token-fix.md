## Problem

The manual release workflow failed in the changelog step with:

- `<!-- generated by git-cliff -->Error: Process completed with exit code 101.`

The failure happened in the `orhun/git-cliff-action` step while generating release changelog content.

## Root Cause

`git-cliff-action` was running without offline mode and attempted to enrich changelog data from GitHub APIs. In this workflow context, metadata responses for commits/pulls returned `404` payloads that did not match the shape expected by git-cliff, which led to a panic (`expected a sequence`) and non-zero exit.

The step also did not explicitly pass `github_token` in the action inputs.

## What Didn't Work

**Attempted state:** `args = "--verbose --tag ..."` without `--offline`.

- **Why it failed:** online metadata enrichment made the step dependent on remote API behavior/shape; the action crashed on unexpected remote response payloads.

## Solution

Keep changelog generation deterministic and local to git history for release execution.

1. Update git-cliff action args in workflow source to force offline mode:
   - `--offline --verbose --tag ${{ steps.tag_version.outputs.new_tag }}`

2. Pass explicit token input to the action:
   - `github_token: ${{ secrets.GITHUB_TOKEN }}`

3. Regenerate workflow YAML from HCL source of truth so generated workflow matches source.

## Verification

Commands run:

- `mise run cinzel github parse --directory ./cinzel --output-directory ./.github/workflows`
- `mise x actionlint@latest -- actionlint`

Observed result:

- Generated `release` workflow now includes `--offline` and explicit `github_token` for `orhun/git-cliff-action`.
- Workflow lint validation passed (non-blocking local `mise` warning about missing `tmux` may still appear in local env).

## Prevention

- Keep release changelog generation in offline mode unless remote metadata is strictly required.
- Always provide explicit action inputs for auth-sensitive steps (`github_token`), rather than relying on implicit defaults.
- Keep release workflow changes in `cinzel/*.hcl` only; regenerate `.github/workflows/*.yaml` and review diff.
- Maintain SHA pinning for marketplace actions and validate with `actionlint` after every workflow update.
